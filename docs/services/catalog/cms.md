# CMS and content platforms

Method legend: webhook, script, polling, email, ci

- WordPress — Possible: yes. Method: webhook/script. Quirks: use plugins or webhooks. Docs: https://developer.wordpress.org/plugins/webhooks/
- Drupal — Possible: yes. Method: webhook/script. Quirks: use rules or webhooks module. Docs: https://www.drupal.org/docs
- Joomla — Possible: yes. Method: webhook/script. Quirks: use plugin or script. Docs: https://docs.joomla.org/
- Ghost — Possible: yes. Method: webhook. Quirks: webhooks in admin. Docs: https://ghost.org/docs/webhooks/
- Contentful — Possible: yes. Method: webhook. Quirks: space-level webhooks. Docs: https://www.contentful.com/developers/docs/concepts/webhooks/
- Strapi — Possible: yes. Method: webhook. Quirks: webhooks in settings. Docs: https://docs.strapi.io/dev-docs/webhooks
- Sanity — Possible: yes. Method: webhook. Quirks: use GROQ webhooks. Docs: https://www.sanity.io/docs/webhooks
- Prismic — Possible: yes. Method: webhook. Quirks: webhook on publish. Docs: https://prismic.io/docs/technologies/webhooks
- Netlify CMS — Possible: yes. Method: webhook/script. Quirks: use build hooks + script. Docs: https://docs.netlify.com/configure-builds/build-hooks/
- Sitecore — Possible: yes. Method: webhook/script. Quirks: use webhooks module or event handlers. Docs: https://doc.sitecore.com/
- Contentstack — Possible: yes. Method: webhook. Quirks: webhooks per stack. Docs: https://www.contentstack.com/docs/developers/webhooks
- DatoCMS — Possible: yes. Method: webhook. Quirks: build hooks. Docs: https://www.datocms.com/docs/content-management-webhooks
- Storyblok — Possible: yes. Method: webhook. Quirks: webhook per space. Docs: https://www.storyblok.com/docs/Guides/webhooks
- Webflow — Possible: yes. Method: webhook. Quirks: form submissions or API triggers. Docs: https://developers.webflow.com/
