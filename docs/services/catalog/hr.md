# HR and people ops

Method legend: webhook, script, polling, email, ci

- Workday — Possible: yes. Method: webhook/script. Quirks: use outbound integrations or reports. Docs: https://community.workday.com/
- BambooHR — Possible: yes. Method: webhook. Quirks: webhooks on specific events. Docs: https://documentation.bamboohr.com/docs/webhooks
- Gusto — Possible: yes. Method: webhook. Quirks: API + webhooks. Docs: https://docs.gusto.com/
- ADP — Possible: yes. Method: webhook/script. Quirks: limited outbound integrations; use API + polling. Docs: https://developers.adp.com/
- Okta — Possible: yes. Method: webhook. Quirks: use event hooks; verify JWT. Docs: https://developer.okta.com/docs/concepts/event-hooks/
- Azure AD (Entra ID) — Possible: yes. Method: webhook. Quirks: use Graph change notifications. Docs: https://learn.microsoft.com/graph/webhooks
- OneLogin — Possible: yes. Method: webhook. Quirks: event webhooks. Docs: https://developers.onelogin.com/api-docs/2/events
- JumpCloud — Possible: yes. Method: webhook. Quirks: use event webhooks. Docs: https://jumpcloud.com/support
- Rippling — Possible: yes. Method: webhook/script. Quirks: limited docs; use API + polling. Docs: https://developers.rippling.com/
- Greenhouse — Possible: yes. Method: webhook. Quirks: use webhooks for candidates/jobs. Docs: https://developers.greenhouse.io/webhooks.html
- Lever — Possible: yes. Method: webhook. Quirks: webhooks per event type. Docs: https://help.lever.co/hc/en-us/articles/205697315-Webhooks
- Workable — Possible: yes. Method: webhook. Quirks: webhook configuration in admin. Docs: https://workable.readthedocs.io/
- Personio — Possible: yes. Method: webhook. Quirks: use webhooks or API polling. Docs: https://developer.personio.de/docs/webhooks
